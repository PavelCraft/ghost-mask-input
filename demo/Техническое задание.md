# Техническое задание: Библиотека ghost-mask-input

---

# 1. Назначение библиотеки

Библиотека предназначена для создания masked input-полей с ghost-placeholder overlay.

Поддерживаются:

* числовые маски;
* текстовые маски;
* смешанные маски;
* пользовательские фильтры;
* пользовательские трансформеры;
* различные стратегии рендера;
* пресеты, задающие цепочку:

```txt
Filter → Transformer → Renderer
```

---

# 2. Архитектурная модель

---

## 2.1. Основные сущности

### InputController

Управляет:

* DOM input;
* событиями ввода;
* raw/formatted state;
* курсором;
* взаимодействием с filter/transformer/renderer.

---

### Filter

Отвечает за:

```txt
- какие символы разрешены ко вводу;
- какие символы считаются валидными пользовательскими данными;
- какие символы в mask являются слотами.
```

Интерфейс:

```js
filter(str)
allowChar(ch)
isValidChar(ch)
isMaskSlot(ch)
```

---

### Transformer

Отвечает за преобразование raw input:

Примеры:

```txt
uppercase
lowercase
trim
slugify
custom normalize
```

Интерфейс:

```js
transform(value)
```

или функция:

```js
(value) => transformedValue
```

---

### Renderer

Отвечает за ghost-placeholder.

Получает:

```txt
mask
formatted
```

Возвращает/рендерит:

```txt
typed
remaining
```

---

### Preset

Preset задаёт цепочку:

```txt
Filter + Transformer + Renderer
```

---

# 3. Источник истины состояния

---

## 3.1. Внутри InputController хранится два значения

```js
rawValue
formattedValue
```

---

### rawValue

Реальные пользовательские данные.

Содержит:

* все символы, введённые пользователем;
* включая символы, совпадающие с separator;
* без автодобавленных mask-разделителей.

---

### formattedValue

Отформатированное представление rawValue по mask.

Используется:

* для отображения в input;
* для ghost renderer.

---

## 3.2. DOM input.value не является source of truth

```txt
input.value = только отображение formattedValue
```

---

# 4. Правила форматирования mask

---

## 4.1. Общая логика

Mask состоит из:

```txt
slot-символов
separator-символов
```

---

### Slot

Определяется через:

```js
filter.isMaskSlot(ch)
```

---

### Separator

Любой символ mask, который:

```js
!filter.isMaskSlot(ch)
```

---

# 5. Правила работы separator

---

## 5.1. Автоподстановка separator

Separator автоматически вставляется в formatted, если:

```txt
после него существует хотя бы один slot,
который будет заполнен.
```

---

## 5.2. Запрет висящих separator

Не допускаются:

```txt
"AA "
"AA-"
"+7 (912) 345-"
```

если separator не special-case.

---

## 5.3. Исключение: закрывающая скобка

---

### Поведение

Если mask содержит:

```txt
(...)
```

то закрывающая скобка появляется сразу после заполнения всех slot внутри скобок.

---

### Пример

```txt
mask: +7 (999) 999-99-99
raw: 7912
formatted: +7 (912)
```

---

### Ограничения

Поддерживаются только:

```txt
невложенные скобки
```

---

# 6. Ambiguous separator handling

---

## 6.1. Требование

Пользователь может вводить символы, совпадающие с separator mask.

---

### Пример

```txt
mask: AA AA
raw: "AA A"
```

валидно.

---

## 6.2. Правило интерпретации

---

### Если введён separator на позиции mask-separator:

Первое совпадение трактуется как:

```txt
масочный separator
```

---

### Повторные подряд такие же separator:

Трактуются как:

```txt
пользовательский ввод
```

---

## 6.3. Пример

---

### Mask

```txt
AA AA
```

---

### Input raw

```txt
AA A
```

---

### Result

```txt
formatted = "AA A"
```

---

### Input raw

```txt
AA  A
```

---

### Result

```txt
formatted = "AA  A"
```

---

### Input raw

```txt
AA   A
```

---

### Result

```txt
formatted = "AA   "
```

(последний A не помещается)

---

## 6.4. Счётчик ambiguous separator

Внутри formatter используется per-separator counter:

```js
{
  ' ': count,
  '-': count,
  ',': count
}
```

---

### Counter ведётся:

```txt
отдельно для каждого separator char
```

---

### Counter сбрасывается:

Когда встречается любой символ raw,
не совпадающий с текущим separator.

````

---

# 7. Поведение Backspace/Delete

---

## 7.1. Удаление происходит по rawValue

---

### Принцип

```txt
Удаляется пользовательский символ,
а не символ formatted-представления.
````

---

## 7.2. Масочные separator напрямую не удаляются

---

### Пример

```txt
mask: AA AA
formatted: "AA  A"
```

Backspace:

```txt
→ "AA A"
```

ещё Backspace:

```txt
→ "AA"
```

---

# 8. Cursor positioning

---

## 8.1. Курсор работает относительно rawValue

---

### Требуется mapping:

```txt
raw index → formatted index
formatted index → raw index
```

---

## 8.2. Cursor всегда перепозиционируется после reformat

---

### При этом:

Пропускаются auto-inserted separators.

---

# 9. Paste behavior

---

## 9.1. Paste вставляет в rawValue

---

### Алгоритм

1. Берётся clipboard text
2. Filter.filter()
3. Вставка в rawValue по raw-index cursor
4. Reformat
5. Cursor reposition

---

## 9.2. Если pasted text содержит separators mask

Они обрабатываются через ambiguous separator logic.

---

# 10. Поддержка повторяющихся separator в mask

---

## Разрешено

