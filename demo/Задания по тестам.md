Я использовал для описания тестовых случаев, которые надо проверить, следующий синтаксис:

- `"abc" -> "abc", "AA"` — ввод текста, результат `typed` и `remaining`
- `backspace (1)` — одно нажатие Backspace, курсор не менял позицию
- `delete (2)` — два нажатия Delete
- Если тесты связаны (используют состояние после предыдущего), они описаны в одном пункте списка.

### 8. Поле «Дублирующиеся разделители» с `data-mask="AA AA--AA..AA"` и `data-preset="text-separated"`

- "ab  cde" -> "ab  c--de", ""
- "ab  c d g " -> "ab  c-- d..g ", ""
- "ab cdefgh" -> "ab cd--ef..gh", ""
- "         " -> "     --  ..  ", ""
- "a.b  .c,d " -> "ab  c--d ", "..AA"
- "abcdefghi" -> "ab cd--ef..gh", ""
- `"ab" -> "ab", " AA--AA..AA"`
- `"abc" -> "ab c", "A--AA..AA"`
- `"abcd" -> "ab cd", "--AA..AA"`
- `"abcde" -> "ab cd--e", "A..AA"`
- `"Г1бdef" -> "Г1 бd--ef", "..AA"`
- `"abcdefg" -> "ab cd--ef..g", "A"`
- `"abcdefgh" -> "ab cd--ef..gh", ""`
- `"ab c"`, курсор после пробела, `"d" -> "ab dc", "--AA..AA"`
- `"ab cd"`, курсор после `c`, `backspace (1) -> "ab d", "A--AA..AA"`
- `"ab cd"`, курсор после пробела, `backspace (1) -> "ac d", "A--AA..AA"`
- `"ab cd--ef"`, курсор после первого дефиса, `delete (1) -> "ab cd--f", "A..AA"` 
-  `"ab cd--ef"`, курсор после первого дефиса, `backspace (1) -> "ab ce--f", "A..AA"` 

Тесты должны выполняться один за другим. Если тестовые кейсы описаны в разных пунктах списка, то они не должны зависеть от состояния друг друга. Напиши тесты с докстрингами на русском языке, и пиши там, что именно тестируется (насколько ты понимаешь). Например, - `"Г1бdef" -> "Г1 бd--ef", "..AA"` - здесь тестируется корректность заполнение поля с учётом разделителей и проверка фильтрации кириллицы. Для тестирования я установил jest и @testing-library/user-event.
